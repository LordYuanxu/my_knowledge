## 阶段一：数据准备 (to_file)

### 1.1 构建下载列表

根据MongoDB中的 `merge_samples` 数据，生成S3文件下载列表。

**生成的文件**:

1. **data_process_merge_sequence_list.json** - S3下载列表
```json
{
    "sequence_dir": [
        {
            "alias": "idx1_69c4d8487fcbe404c52357c3--L1MKL1607686--L_PN0_3.R1.raw.fastq.gz",
            "file_path": "s3nb4://datasplit/2025/LibrarySplit/.../xxx.R1.raw.fastq.gz",
            "indx": 1
        },
        {
            "alias": "idx2_69c4d8487fcbe404c52357c3--L1MKL1607686--L_PN0_3.R1.raw.fastq.gz",
            "file_path": "s3nb4://datasplit/2025/LibrarySplit/.../xxx.R1.raw.fastq.gz",
            "indx": 2
        }
    ]
}
```

### 1.2 构建合并参数

根据产品类型、样本信息生成合并参数配置。

**data_process_merge_info.json**:
```json
{
    "69c4d8487fcbe404c52357c3_raw_2": {
        "data_type": "rna_raw",
        "file_format": "fastq",
        "merge_id": "69c4d8487fcbe404c52357c3_raw_2",
        "source_path": "/path/to/file1.fastq.gz;/path/to/file2.fastq.gz",
        "dest_basename": "69c4d8487fcbe404c52357c3--L1MKL1607686--L_PN0_3.R2.raw.fastq.gz",
        "sample_name": "L_PN0_3"
    }
}
```

**关键字段说明**:
| 字段            | 说明                                                      |
| --------------- | --------------------------------------------------------- |
| `data_type`     | 数据类型：rna_raw/rna_clean/dna_raw/dna_clean/mirna_raw等 |
| `file_format`   | 文件格式：fastq/bam                                       |
| `source_path`   | 多个源文件路径，用分号分隔                                |
| `dest_basename` | 合并后的目标文件名                                        |

## 阶段三：数据合并 (Workflow)

### 3.1 合并流程图

```
开始
  │
  ▼
┌────────────────┐
│ 读取合并参数    │◀── data_process_merge_info.json
│  (info.json)   │
└───────┬────────┘
        │
        ▼
┌────────────────┐
│ 判断文件格式    │
└───────┬────────┘
        │
        ├───▶ fastq ──▶ 使用 cat 合并
        │
        └───▶ bam ────▶ 使用 pbmerge 合并
        │
        ▼
┌────────────────┐
│ 多样性产品？    │
└───────┬────────┘
        │
        ├───▶ 是 ────▶ 序列重命名
        │
        └───▶ 否 ────▶ 跳过
        │
        ▼
┌────────────────┐
│ 文件校验        │◀── fastq检查文件大小
└───────┬────────┘
        │
        ▼
      结束
```

### 3.2 合并工具选择

| 文件格式  | 合并工具  | 资源申请        | 说明           |
| --------- | --------- | --------------- | -------------- |
| **fastq** | `cat`     | 1 CPU, 3G 内存  | 直接拼接       |
| **bam**   | `pbmerge` | 4 CPU, 10G 内存 | PacBio官方工具 |

### 3.3 特殊处理

#### 多样性产品序列重命名

多样性产品合并后需要对序列名进行重命名，确保不同lane的数据可以区分。

**处理逻辑**:
```python
# 伪代码
if product_type == "多样性":
    for record in fastq:
        record.id = f"{lane_name}_{record.id}"
        write(record)
```

#### 文件校验

**fastq文件**:
```bash
# 合并前大小总和 = 合并后大小
ls -l file1.fastq.gz file2.fastq.gz
# 总大小应该等于 merged.fastq.gz
```

**bam文件**:
- 暂无自动校验
- 合并成功即认为没有问题

## 关键决策点

| 决策点     | 判断条件     | 分支处理                         |
| ---------- | ------------ | -------------------------------- |
| 合并工具   | file_format  | fastq → cat; bam → pbmerge       |
| 序列重命名 | product_type | 多样性 → 重命名; 其他 → 跳过     |
| 文件校验   | file_format  | fastq → 检查大小; bam → 不检查   |
| raw75      | data_type    | miRNA → 处理raw75; 其他 → 不处理 |

---

## 注意事项

1. **历史原因问题**:
   - 纯文库没有 `library_type_name` (文库名)
   - 三级产品名可能没有传递

2. **三代测序**:
   - 可能使用bam格式而非fastq
   - 需要使用pbmerge工具

3. **并行处理**:
   - 样本级别可以并行
   - 通过 `merge_parallel` 模块实现
