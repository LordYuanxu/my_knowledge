## 上传方式

### Samba 共享上传（推荐）

适用于实验室测序仪直接上传或手动上传数据。

#### 连接信息

| 配置项       | 内容                      |
| ------------ | ------------------------- |
| **网络地址** | `\\172.16.11.73\三代菌鉴` |
| **用户名**   | `mjdm`                    |
| **密码**     | `123456`                  |
| **协议**     | SMB/CIFS                  |

#### Windows 连接步骤

1. **打开文件资源管理器**
   - 按 `Win + E` 打开资源管理器

2. **连接到共享**
   - 在地址栏输入：`\\172.16.11.73\三代菌鉴`
   - 按回车键

3. **输入凭据**
   ```
   用户名: mjdm
   密码: 123456
   ```

4. **上传文件**
   - 将测序数据文件复制到共享目录中
   - 确保文件命名符合规范（见下文）

#### Linux/Mac 连接步骤

```bash
# 使用 smbclient
smbclient //172.16.11.73/三代菌鉴 -U mjdm

# 或使用 mount
sudo mount -t cifs //172.16.11.73/三代菌鉴 /mnt/samba -o username=mjdm,password=123456
```

## 文件命名规范

上传的文件必须遵循以下命名规范，以便系统自动识别样本信息。

### 必需文件类型

| 文件类型          | 命名格式示例                                        | 说明         |
| ----------------- | --------------------------------------------------- | ------------ |
| **ab1文件**       | `{测序编号}-{majorbio_number}.ab1`                  | 原始测序峰图 |
| **consensus序列** | `{测序编号}-{majorbio_number}.consensus.fasta`      | 组装序列     |
| **覆盖度图**      | `{测序编号}-{majorbio_number}.coverage.png`         | 覆盖图       |
| **读长分布图**    | `{测序编号}-{majorbio_number}_read_length_peak.png` | 长度分布图   |
| **突变位点**      | `{测序编号}-{majorbio_number}.filt.var.xls`         | 变异信息     |

### 可选文件类型

| 文件类型      | 命名格式示例                         | 说明                        |
| ------------- | ------------------------------------ | --------------------------- |
| **BAM文件**   | `{测序编号}-{majorbio_number}.bam`   | 比对文件（如无fastq则必需） |
| **FASTQ文件** | `{测序编号}-{majorbio_number}.fastq` | 原始测序数据                |

### 命名示例

```
样本信息：
- majorbio_number: RLC1301763-2
- sample_name: S7
- 测序编号: S22604013025

对应文件：
S22604013025-RLC1301763-2_1.ab1
S22604013025-RLC1301763-2.consensus.fasta
S22604013025-RLC1301763-2.coverage.png
S22604013025-RLC1301763-2_read_length_peak.png
S22604013025-RLC1301763-2.var.xls
S22604013025-RLC1301763-2.bam
```

## 常见问题

### Q: 文件上传后系统无法识别？

**A:** 检查以下几点：
1. 文件名是否符合规范（特别是`majorbio_number`部分）
2. 文件是否在正确的目录下
3. 是否包含必需的ab1和consensus文件

### Q: 缺少fastq文件可以上传吗？

**A:** 可以。如果没有fastq但有bam文件，系统会自动将bam转换为fastq用于Kraken分析。

### Q: 上传后多久能在系统中看到？

**A:** 文件上传后立即可见，但需要在系统中创建任务并投递后才能触发分析流程。
