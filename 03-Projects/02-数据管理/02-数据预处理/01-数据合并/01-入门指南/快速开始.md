## 概述

数据合并是数据预处理的重要模块，用于将同一个样本在不同lane（或不同测序批次）的测序数据合并成一个完整的文件。

**支持的产品类型：**
- 🧬 非多样性测序
- 🦠 多样性（16S/ITS等）
- 🧫 单细胞测序
- 🎯 时空转录组
- 🧪 miRNA
- 📊 三代测序

**支持的数据格式：**
- `fastq.gz` - 标准压缩fastq格式
- `bam` - 三代测序BAM格式

---

## 了解核心流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 释放单  │───▶│ 投递任务 │───▶│ S3下载  │───▶│ 数据合并 │───▶│ 结果上传 │
│ (MongoDB)│   │         │   │         │   │         │   │         │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
                              支持格式:        合并工具:
                              - fastq.gz      - cat
                              - bam           - pbmerge
```

**核心步骤：**
1. **创建释放单** - 在系统中创建包含样本信息的释放单
2. **投递任务** - 通过API或Web界面启动合并流程
3. **下载数据** - 从S3下载各lane的数据到本地
4. **合并数据** - 使用cat/pbmerge工具合并文件
5. **上传结果** - 将合并后的文件上传到S3
