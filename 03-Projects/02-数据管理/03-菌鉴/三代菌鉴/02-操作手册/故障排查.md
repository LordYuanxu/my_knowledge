## 一、任务投递问题

### 1.1 投递失败："main_id not found"

**现象：**
```json
{
    "info": "main_id not found",
    "success": false
}
```

**排查步骤：**

1. **检查ID是否存在**
   ```javascript
   db.strain_identification_third.findOne({"_id": ObjectId("69cdbf908e08c37e72effc57")})
   ```

2. **检查ID字符串格式**
   - 确保是24位十六进制字符串
   - 不要有多余的空格

**解决方案：**
- 确认数据已正确插入MongoDB
- 检查ID字符串是否正确

---

### 1.2 投递失败："文件匹配失败"

**现象：** 任务投递成功，但工作流立即失败

**排查步骤：**

1. **检查文件是否上传**
   ```bash
   ls /mnt/clustre/users/sanger-dev/rsync_data/third_strain_identification/2026/0402/
   ```

2. **检查文件名规范**
   - 确认包含 `majorbio_number`
   - 确认文件后缀正确

3. **检查样本信息匹配**
   ```javascript
   db.strain_identification_sample_third.find({"main_id": ObjectId("...")})
   ```

**解决方案：**
- 重新上传符合规范的文件
- 检查 `majorbio_number` 是否正确

## 二、工作流运行问题

### 2.1 任务状态卡在 "start"

**现象：** 任务状态长时间为 `start`，没有进展

**排查步骤：**

1. **检查SLURM队列**
   ```bash
   squeue -u sanger-dev | grep StrainIdentification
   ```

2. **检查工作流日志**
   ```bash
   tail -100 /mnt/clustre/users/sanger-dev/wpm2/workspace/.../logs/*.log
   ```

3. **检查WPM平台**
   - 登录 WPM 查看任务详情
   - 查看是否有失败的步骤

**可能原因：**
- SLURM队列拥堵
- 资源不足
- 依赖任务失败

---

### 2.2 BLAST步骤失败

**现象：** 工作流在BLAST步骤报错

**排查步骤：**

1. **检查输入文件**
   ```bash
   ls -lh /path/to/consensus.fasta
   cat /path/to/consensus.fasta
   ```

2. **检查BLAST日志**
   ```bash
   grep -i error /path/to/blast_logs/*.log
   ```

3. **检查NCBI数据库连接**
   - 确认数据库可用
   - 检查网络连接

**常见原因：**
- consensus序列为空
- NCBI数据库不可用
- 磁盘空间不足

---

### 2.3 Kraken步骤失败

**现象：** 工作流在Kraken步骤报错

**排查步骤：**

1. **检查输入文件**
   ```bash
   # 检查fastq是否存在
   ls -lh /path/to/*.fastq
   
   # 检查bam是否存在（如果没有fastq）
   ls -lh /path/to/*.bam
   ```

2. **检查bam转fastq过程**
   ```bash
   # 查看转换日志
   grep -i "bam\|fastq" /path/to/logs/*.log
   ```

**常见原因：**
- fastq和bam都不存在
- bam转fastq失败
- Kraken数据库问题

## 三、分析结果问题

### 3.1 结果文件缺失

**现象：** 任务状态为 `end`，但结果文件不完整

**排查步骤：**

1. **检查S3路径**
   ```bash
   aws s3 ls s3nb4://datasplit/.../result.zip
   ```

2. **检查工作目录输出**
   ```bash
   ls -lh /path/to/output/
   ```

3. **检查打包日志**
   ```bash
   grep -i "zip\|pack" /path/to/logs/*.log
   ```

---

### 3.2 BLAST结果为空

**现象：** blast结果文件存在，但内容为空

**排查步骤：**

1. **检查consensus序列**
   ```bash
   cat /path/to/consensus.fasta
   ```

2. **手动运行BLAST测试**
   ```bash
   blastn -query consensus.fasta -db nt -out test.blast
   ```

**常见原因：**
- consensus序列质量太差
- 序列包含过多N
- 非目标物种序列

---

### 3.3 污染判断异常

**现象：** `quality_rank` 标记为污染，但实际样本正常

**排查步骤：**

1. **检查var文件**
   ```bash
   cat /path/to/var.xls
   ```

2. **检查判断参数**
   - 查看 `var_judge` 工具的配置参数

---

## 四、数据库导出问题

### 4.1 导表失败

**现象：** 分析完成，但MongoDB中无数据更新

**排查步骤：**

1. **检查导表日志**
   ```bash
   grep -i "import\|database" /path/to/logs/*.log
   ```

2. **检查MongoDB连接**
   ```javascript
   db.runCommand({ping: 1})
   ```

3. **检查数据格式**
   - 确认导表数据格式正确
