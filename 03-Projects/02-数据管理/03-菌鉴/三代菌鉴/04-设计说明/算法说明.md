## 一、污染判定算法

### 1.1 原理

基于突变位点（SNP）分析判断样本是否存在污染。

**核心思想：**
- 纯菌样本的突变位点应该比较一致
- 污染样本会出现杂合位点（多个碱基）
- 通过统计杂合位点比例判断是否污染

### 1.2 输入数据

`var.xls` 文件，包含突变位点信息：

| 字段     | 说明         |
| -------- | ------------ |
| Position | 突变位置     |
| Ref      | 参考碱基     |
| Alt      | 突变碱基     |
| Depth    | 测序深度     |
| AF       | 等位基因频率 |

### 1.3 判定逻辑

```python
# 伪代码
杂合位点比例 = 杂合位点数 / 总位点数

if 杂合位点比例 > 阈值:
    quality_rank = "污染"
else:
    quality_rank = "正常"
```

**判定标准：**
- 杂合位点：AF（等位基因频率）在 0.2-0.8 之间
- 阈值：根据经验设定（通常 5%-10%）

### 1.4 输出

- `quality_rank`: "正常" 或 "污染"
- 可能跳过部分分析步骤

## 二、BLAST比对算法

### 2.1 原理

BLAST (Basic Local Alignment Search Tool) 是一种序列相似性搜索工具。

**核心思想：**
- 将查询序列与数据库中的序列进行局部比对
- 找到相似度最高的匹配序列
- 评估比对的显著性

### 2.2 输入输出

**输入：**
- `consensus.fasta` - 样本的一致序列

**输出：**
- BLAST比对结果表，包含多个匹配记录

### 2.3 关键指标

| 指标          | 说明     | 判断标准                 |
| ------------- | -------- | ------------------------ |
| **Bit-score** | 比对得分 | 越高越好                 |
| **E-value**   | 期望值   | 越接近0越好 (< 1e-5)     |
| **Identity**  | 相似度   | 百分比，越高越好 (> 97%) |
| **Gaps**      | 空缺数   | 越少越好                 |
| **Length**    | 比对长度 | 覆盖查询序列的比例       |

### 2.4 BLAST输出字段详解

```
┌────────────────────────────────────────────────────────────────┐
│ qseqid    Query序列ID      - 样本编号                          │
│ sseqid    Subject序列ID    - 数据库序列ID                      │
│ pident    相似度百分比     - 100%表示完全相同                  │
│ length    比对长度         - 匹配区域的长度                    │
│ mismatch  错配数           - 不匹配的碱基数                    │
│ gapopen   空缺开放数       - 空位数量                          │
│ qstart    Query起始位置    - 查询序列匹配起始                  │
│ qend      Query结束位置    - 查询序列匹配结束                  │
│ sstart    Subject起始位置  - 目标序列匹配起始                  │
│ send      Subject结束位置  - 目标序列匹配结束                  │
│ evalue    期望值           - 随机匹配的概率，越小越显著        │
│ bitscore  比特得分         - 比对的显著性评分                  │
└────────────────────────────────────────────────────────────────┘
```

## 三、Kraken分类算法

### 3.1 原理

Kraken 是一种快速的序列分类工具，基于 k-mer 匹配。

**核心思想：**
- 将DNA序列分割为固定长度(k)的短片段(k-mer)
- 与预构建的分类数据库匹配
- 根据k-mer的物种归属进行分类

### 3.2 k-mer概念

```
序列: ATGCGATCGATCGA
k=8

k-mers:
- ATGCGATC
- TGCGATCG
- GCGATCGA
- CGATCGAT
- GATCGATC
- ATCGATCG
...
```

### 3.3 分类逻辑

```
┌─────────────────────────────────────────────────────────┐
│                    Kraken分类流程                        │
├─────────────────────────────────────────────────────────┤
│  1. 读取序列                                             │
│  2. 分割为k-mers                                        │
│  3. 查询数据库 - 每个k-mer对应一个物种/分类单元(LCA)     │
│  4. 统计投票 - 根据匹配的k-mers进行投票                  │
│  5. 确定分类 - 选择票数最多的分类                        │
└─────────────────────────────────────────────────────────┘
```

**LCA (Lowest Common Ancestor):**
- 如果一个k-mer匹配多个物种，取最近共同祖先
- 例如：匹配 E.coli 和 S.typhimurium → LCA 为 Enterobacteriaceae

### 3.4 输入输出

**输入：**
- `fastq` 文件（首选）或 `bam` 转换后的 `fastq`

**输出：**
- 分类结果表

| 字段          | 说明                    |
| ------------- | ----------------------- |
| C/U           | Classified/Unclassified |
| ReadName      | 读段名称                |
| TaxID         | 物种Taxonomy ID         |
| Length        | 读段长度                |
| k-mer分类详情 | 每个k-mer的分类信息     |

## 四、算法比较

| 特性         | BLAST        | Kraken       |
| ------------ | ------------ | ------------ |
| **原理**     | 序列比对     | k-mer匹配    |
| **速度**     | 慢           | 快           |
| **准确性**   | 高（长序列） | 中等         |
| **数据库**   | NCBI NT      | Kraken标准库 |
| **适用场景** | 精确鉴定     | 快速筛查     |
| **输出**     | 相似度得分   | 分类置信度   |

## 五、结果整合

### 5.1 综合判定逻辑

```
┌─────────────────────────────────────────────────────────────┐
│                      结果整合                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   BLAST结果 ──┐                                             │
│               ├──▶ 综合判定 ──▶ 最终菌种鉴定结果           │
│   Kraken结果 ──┘                                             │
│                                                             │
│   判定规则：                                                 │
│   1. 优先参考BLAST结果（更准确）                            │
│   2. Kraken作为辅助验证                                     │
│   3. 不一致时人工审核                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 结果输出

最终输出 `identify.xls`：

| 字段           | 说明                     |
| -------------- | ------------------------ |
| Sample         | 样本名                   |
| MajorbioNumber | 美吉编号                 |
| Species        | 鉴定物种                 |
| Confidence     | 置信度                   |
| Method         | 鉴定方法（BLAST/Kraken） |
