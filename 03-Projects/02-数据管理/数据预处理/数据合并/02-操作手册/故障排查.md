## 一、任务投递问题

### 1.1 投递失败："释放单不存在"

**现象：**
```json
{
    "info": "释放单不存在",
    "success": false
}
```

**排查步骤：**

1. **检查释放单ID**
   ```javascript
   db.data_process_merge.findOne({"fx_id": 690575})
   ```

2. **检查释放单状态**
   - 确认释放单已创建
   - 确认释放单已拆分完成

**解决方案：**
- 确认fx_id是否正确
- 等待拆分完成后再投递

---

### 1.2 投递失败："释放单无样本"

**现象：** 任务投递成功，但工作流立即失败

**排查步骤：**

1. **检查样本表**
   ```javascript
   db.data_process_merge_detail.find({"fx_id": 690575})
   ```

2. **检查merge_samples**
   ```javascript
   db.data_process_merge_detail.findOne({"fx_id": 690575}).merge_samples
   ```

**解决方案：**
- 确认释放单已完成拆分
- 确认样本数据已正确关联

## 二、S3下载问题

### 2.1 to_file阶段失败

**现象：** 工作流在to_file阶段报错，无法下载数据

**排查步骤：**

1. **检查生成的下载列表**
   ```bash
   cat work_dir/remote_input/data_process_merge_sequence_list.json
   ```

2. **检查S3文件是否存在**
   ```bash
   aws s3 ls s3nb4://datasplit/path/to/file.fastq.gz
   ```

3. **检查S3权限**
   ```bash
   # 测试读取权限
   aws s3 cp s3nb4://datasplit/path/to/test.file ./test.file
   ```

**常见原因：**
- S3文件不存在
- 源数据已被删除或移动
- S3访问权限问题

---

### 2.2 下载超时

**现象：** 下载过程中断，日志显示超时

**排查步骤：**

1. **检查网络连接**
   ```bash
   ping s3nb4-endpoint
   ```

2. **检查文件大小**
   ```bash
   aws s3 ls --human-readable s3nb4://datasplit/path/to/file.fastq.gz
   ```

**解决方案：**
- 大文件可能需要增加超时时间
- 检查网络带宽

## 三、数据合并问题

### 3.1 合并失败：文件不存在

**现象：** data_merge工具报错，找不到输入文件

**排查步骤：**

1. **检查下载的文件**
   ```bash
   ls -lh work_dir/remote_input/sequence_dir/
   ```

2. **检查merge_info.json**
   ```bash
   cat work_dir/remote_input/data_process_merge_info.json
   ```

3. **检查路径是否正确**
   - 确认 `source_path` 中的路径与实际下载路径一致

---

### 3.2 合并失败：文件格式错误

**现象：** 合并工具报错，文件格式不正确

**排查步骤：**

1. **检查文件格式**
   ```bash
   # 检查fastq格式
   zcat file.fastq.gz | head -4
   
   # 检查bam格式
   samtools view -H file.bam
   ```

2. **检查文件完整性**
   ```bash
   # 检查gzip完整性
   gzip -t file.fastq.gz
   ```

**常见原因：**
- 下载不完整
- 文件损坏
- 格式标识错误（实际为bam但标记为fastq）

---

### 3.3 合并后文件校验失败

**现象：** fastq合并后文件大小校验不通过

**排查步骤：**

1. **检查源文件大小**
   ```bash
   ls -l source1.fastq.gz source2.fastq.gz
   # 记录总大小
   ```

2. **检查合并后文件大小**
   ```bash
   ls -l merged.fastq.gz
   ```

3. **手动验证**
   ```bash
   # 解压后检查行数
   zcat source1.fastq.gz source2.fastq.gz | wc -l
   zcat merged.fastq.gz | wc -l
   ```

**常见原因：**
- 合并过程中断
- 磁盘空间不足导致写入不完整

## 四、结果上传问题

### 4.1 上传S3失败

**现象：** 合并完成但无法上传到S3

**排查步骤：**

1. **检查S3路径权限**
   ```bash
   aws s3 ls s3nb4://datasplit/target/path/
   ```

2. **检查磁盘空间**
   ```bash
   df -h
   ```

3. **手动测试上传**
   ```bash
   aws s3 cp test.file s3nb4://datasplit/test.file
   ```

## 五、数据库导出问题

### 5.1 导表失败

**现象：** 分析完成，但MongoDB中无数据更新

**排查步骤：**

1. **检查导表日志**
   ```bash
   grep -i "import\|database\|error" work_dir/logs/*.log
   ```

2. **检查MongoDB连接**
   ```javascript
   db.runCommand({ping: 1})
   ```

3. **检查path字段格式**
   - 确认导表数据格式正确
   - 检查路径是否为字符串格式

### 6.1 多样性产品重命名失败
