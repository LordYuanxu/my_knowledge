# Go语言代码测试

---

## 基础测试结构

### 文件命名

- 测试文件必须以 `_test.go` 结尾，如 `configloader_test.g
- 与被测代码放在**同一包**（package）下

### 最小测试示例

```go
package configloader

import "testing"

// 函数名必须以 Test 开头，参数必须是 *testing.T
func TestResolvePath(t *testing.T) {
    // 调用被测函数
    result := resolvePath("/etc/config.yml")
    
    // 验证结果（Go 没有内置断言，手动判断）
    if result != "/etc/config.yml" {
        // t.Errorf 不会停止测试，t.Fatalf 会立即终止
        t.Errorf("resolvePath() = %s, want %s", result, "/etc/config.yml")
    }
}
```

### 运行测试

```bash
go test ./...                    # 运行所有测试
go test -v ./...                 # 详细输出（显示每个测试函数名）
go test -run TestResolvePath     # 只运行特定测试
```

## 2. 表驱动测试（Go 推荐模式）

当有多个测试用例时，使用**表驱动测试**：

```go
func TestResolvePath(t *testing.T) {
    tests := []struct {
        name     string
        input    string
        expected string
    }{
        {"绝对路径", "/etc/config.yml", "/etc/config.yml"},
        {"空路径优先环境变量", "", "/env/path/config.yml"},
        {"相对路径", "config.yml", "/current/dir/config.yml"},
    }

    for _, tt := range tests {
        // t.Run 创建子测试，可单独运行
        t.Run(tt.name, func(t *testing.T) {
            // 设置环境变量（仅限此子测试）
            if tt.input == "" {
                t.Setenv("APP_CONFIG_PATH", "/env/path/config.yml")
            }
            
            got := resolvePath(tt.input)
            if got != tt.expected {
                t.Errorf("resolvePath() = %v, want %v", got, tt.expected)
            }
        })
    }
}
```

**运行单个子测试**

```bash
go test -run "TestResolvePath/绝对路径"
```

## 使用 testify 简化断言（可选但常用）

虽然标准库够用，但 `testify` 提供了类似其他语言的断言风格：

```go
import "github.com/stretchr/testify/assert"

func TestResolvePath(t *testing.T) {
    got := resolvePath("config.yml")
    
    // 一行断言，自动格式化错误信息
    assert.Equal(t, "/current/dir/config.yml", got)
    assert.Contains(t, got, "config.yml")
    assert.NotEmpty(t, got)
    
    // 使用 require 会立即终止测试（类似 Fatalf）
    require.NoError(t, err)
}
```

## 性能测试

测试函数以 `Benchmark` 开头，参数为 `*testing.B`：

```go
func BenchmarkReplaceEnvVars(b *testing.B) {
    content := "host: ${DB_HOST}, port: ${DB_PORT}"
    
    // 重置计时器（排除初始化时间）
    b.ResetTimer()
    
    // 循环 b.N 次，Go 自动调整 N 直到稳定
    for i := 0; i < b.N; i++ {
        replaceEnvVars(content)
    }
}
```

运行：

```bash
go test -bench=.            # 运行所有 benchmark
go test -bench=. -benchmem  # 显示内存分配情况
```

## 示范测试

以 `Example` 开头的函数，既作为测试，又生成文档：

```go
func ExampleLoader_Load() {
    loader := &Loader{}
    err := loader.Load(&LoadOptions{Path: "testdata/config.yml"})
    if err != nil {
        fmt.Println(err)
        return
    }
    
    cfg, _ := loader.Get("database")
    fmt.Println(cfg["host"])
    
    // Output: localhost
}
```

## 测试辅助函数（Setup/Teardown）

Go 没有内置的 Setup/Teardown，用普通函数 + `t.Cleanup`：

```go
func setupTempConfig(t *testing.T) string {
    tmpDir := t.TempDir()  // Go 1.15+ 自动创建临时目录，测试结束自动清理
    
    configPath := filepath.Join(tmpDir, "config.yml")
    os.WriteFile(configPath, []byte("key: value"), 0644)
    
    return configPath
}

func TestLoadConfig(t *testing.T) {
    path := setupTempConfig(t)
    
    // 测试逻辑
    loader := &Loader{}
    err := loader.Load(&LoadOptions{Path: path})
    
    if err != nil {
        t.Fatal(err)
    }
    
    // t.TempDir() 和 t.Cleanup 会在测试结束后自动清理文件
}
```
