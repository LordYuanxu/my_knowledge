# 配置

## 基础配置

```bash
# 查看配置
## 查看基本配置
git config user.name
git config user.email
## 或看完整配置
git config --list | grep user

# 修改配置
git config --global user.name "你的真实名字"
git config --global user.email "你的邮箱@example.com"
```

## 配置远程仓库

```bash
# 远程配置
git remote -v

# 添加远程(本地和远程关联起来)
git remote add origin git@git.majorbio.com:bxy200502367/mjdm-cli.git
git remote add origin git@github.com:LordYuanxu/mjdm-cli.git

# 验证链接有没有问题
ssh -T git@github.com

# 拉取
git pull origin main
```

##  查看看信息

```bash
# 查看所有出现过的作者（去重）
git log --format="%an <%ae>" | sort -u
# 创建 mailmap 文件（不要直接用 `<()`）
cd /path/to/your-project

# 创建 mailmap 文件
cat > mailmap.txt << 'EOF'
yuan xu <yuan.xu@majorbio.com> Ubuntu <ubuntu@localhost.localdomain>
yuan xu <yuan.xu@majorbio.com> yuan.xu <yuan.xu@majorbio.com>
EOF

# 安装（如果没有）
pip install git-filter-repo

# 执行（--force 表示接受在非全新仓库上操作）
git filter-repo --mailmap mailmap.txt --force
# 直接 force push（如果 main 不是保护分支）
git push origin main --force-with-lease
```

# 合并

## 本地合并分支后推送到远程

```bash
# 1. 切换到目标分支（要合并进去的分支）
git checkout main

# 2. 拉取远程最新代码（防止冲突）
git pull origin main

# 3. 合并源分支
git merge feat/agent-task-system

# 4. 推送到远程
git push origin main

# 删除本地已合并的分支
git branch -d feat/agent-task-system

# 删除远程的该分支
git push origin --delete feat/agent-task-system
```


```bash
# 修改最后一次提交的作者
git commit --amend --reset-author --no-edit

# 然后强制推送覆盖远程（谨慎！）
git push origin main --force-with-lease

# 安装 git-filter-repo（如果没有）
pip install git-filter-repo

# 进入项目目录
cd /path/to/your-project

# 批量替换所有历史提交的作者和提交者
git filter-repo --mailmap <(echo 'ubuntu <ubuntu@localhost> yuan xu <yuan.xu@majorbio.com>')
git push --all origin --force-with-lease
```
