# 文件夹手动删除或者清理
## 特定文件之外的文件(如工作目录只保留data.json文件)
```bash
find . -type f ! -name "data.json" -exec rm -f {} \;
# linux 如何rm删除某个目录下除了data.json外的所有文件和文件夹
## 方法 1：使用 `find` + `rm`（推荐）
cd /path/to/directory  # 进入目标目录
find . -mindepth 1 -maxdepth 1 ! -name "data.json" -exec rm -rf {} +
## 方法 2：使用 `find` + `xargs`
cd /path/to/directory  # 进入目标目录
find . -mindepth 1 -maxdepth 1 ! -name "data.json" | xargs rm -rf
## 方法 3：使用 `ls` + `grep` + `xargs`（不推荐，仅适用于简单场景）
cd /path/to/directory  # 进入目标目录
ls -A | grep -v "data.json" | xargs rm -rf
```

## 删除某个日期的文件夹
```bash
# 先预览
find /mnt/upload/R21016100250027/DNBSEQ-T7+ \
 -maxdepth 1 \
 -type d \
 ! -name "Info" \
 ! -name "R21016100250027" \
 ! -newermt "2026-01-01" \
 -exec lsd -ld {} \;

# 执行删除(务必先预览确认!)
find /mnt/upload/R21016100250027/DNBSEQ-T7+ \
 -maxdepth 1 \
 -type d \
 ! -name "Info" \
 ! -name "R21016100250027" \
 ! -newermt "2026-01-01" \
 -exec bash -c 'echo "✅ Deleting folder: $0" && rm -rf "$0"' {} \;
```

## 手动删除s3上的文件夹
**使用的情况:**
1. 生成新的workflow而不是重运行，原来的工作流废弃，但是有数据上传s3了
2. 需要主动删除，只是临时性的工作

```
# 使用mc工具,先dry-run模式预览内容,避免误删
mc rm --recursive --force --dry-run s3nb3/dataprocess/2026/DataProcessMerge/ningbo/20260326/DataProcessMerge_FX2026032500983_683346_20260326_171006/
# 实际删除
mc rm --recursive --force s3nb3/dataprocess/2026/DataProcessMerge/ningbo/20260326/DataProcessMerge_FX2026032500983_683346_20260326_171006/
# 如果是涉及数据流转，和下游确认一下更新，因为可能原来的路径已经传递下去了
```
